= Angkor Kotlin API Backend

====
This is the getting started readme for the *Angkor Kotlin Module* in the `/kotlin` sub-folder,
for general Information on the Project please refer to the readme in the Project root!
====

== Project Overview

Angkor is a RESTful API backend built with **Spring Boot 3.x** and **Kotlin** for managing travel-related content. It provides CRUD operations for various entities including photos, places, tours, dishes, events, and more.

=== Key Features

* **Entity Management** — Full CRUD support for 12+ domain entities (Photos, Places, Tours, Dishes, Events, Notes, Links, etc.)
* **Geolocation Support** — Integration with OpenStreetMap (Nominatim) for reverse geocoding and location search
* **Media Processing** — Photo and video management with feed import capabilities (RSS/JSON)
* **Tour Management** — Track planned vs. recorded tours with GPX import/export
* **Tagging System** — Flexible tagging with PostgreSQL `hstore` and array support
* **Event Sourcing** — Kafka-based CloudEvents for audit trail and system events
* **OAuth2 Security** — AWS Cognito integration with role-based access control (USER/ADMIN)
* **Caching** — Spring Cache abstraction with configurable TTL
* **Metrics** — Prometheus metrics export via Spring Boot Actuator
* **Database** — PostgreSQL with Flyway migrations, JPA entities using Hibernate + Hypersistence Utils

=== Architecture Highlights

* Kotlin data classes for JPA entities with type-safe nullability
* Generic `AbstractEntityService<ET, EST, ID>` and `AbstractEntityController<ET, EST, ID>` base classes
* Repository pattern with `Searchable<T>` and `AuthScopeSupport<T>` interfaces
* Repository IDs are non-nullable; use `!!` for null checks when compiler validates safety
* Transactional service layer with `@Transactional(readOnly = true)` for reads
* Security enforced via `SecurityUtils.allowedAuthScopes()` filtering
* Comprehensive test coverage with JUnit 5, Mockito Inline, WireMock, and ArchUnit

=== Tech Stack

* **Language** — Kotlin (targeting Java 21)
* **Framework** — Spring Boot 3.x, Spring Security, Spring Data JPA
* **Database** — PostgreSQL 12+ with Flyway migrations
* **Messaging** — Apache Kafka with CloudEvents (cloudevents-kafka)
* **Testing** — JUnit 5, Mockito Inline, WireMock, GreenMail, ArchUnit
* **Build** — Gradle 8.x with Version Catalog (`libs.versions.toml`)
* **Monitoring** — Spring Boot Actuator + Prometheus
* **Container** — Docker support with multi-stage builds

== Install Postgres

* Install local Postgres DB. As of this writing we're using Version 16!
See https://medium.com/@viviennediegoencarnacion/getting-started-with-postgresql-on-mac-e6a5f48ee399[Getting started with Postgres on Mac] for a more detailed tutorial
+
----
$ brew install postgresql@16
postgresql@16 16.11 is already installed and up-to-date.
If you need to have postgresql@16 first in your PATH, run:
  echo 'export PATH="/usr/local/opt/postgresql@16/bin:$PATH"' >> $HOME/.bash_profile
To have launchd start postgresql@16 now and restart at login:
  brew services start postgresql@16
Or, if you don't want/need a background service you can just run:
  pg_ctl -D /usr/local/var/postgresql@16 start
$ pg_ctl -D /usr/local/var/postgres  init
Success. You can now start the database server using:

pg_ctl -D /usr/local/var/postgres -l logfile start
$ psql postgres

Type "help" for help.

postgres=# \du

[CAUTION]
====
Above SQL snippets assume you're using default db settings as per `application.yml` default profile.
If you want to use different DB name or users, or use an existing remote DB, you use `config/application.properties` to
override without putting the custom settings to source control!
====

* Edit the `application.properties` in the `config/` folder (you can use the `*.tmpl` file in the same dir as a template)  to configure credentials for local external services such as *Postgres* and *Kafka*. Those will take precedence at runtime, but make sure you don't put this file under version control!
+
----
~/angkor/api$ cp config/application.properties.tmpl config/application.properties
~/angkor/api$ code config/application.properties
----

== CloudEvents

* https://dzone.com/articles/cloud-events-kafka-binding-with-spring[DZone: Cloud-Events Kafka Binding With Spring] (https://github.com/wkennedy/spring-kafka-cloud-events[GitHub repo]) very good article, selected quotes below:
** The CloudEvents Kafka protocol binding provides two content modes for transferring data.
*** Structured: event metadata attributes and event data are placed into the Kafka message value section using an event format.
*** Binary: the value of the event data MUST be placed into the Kafka message's value section as-is, with the content-type header value declaring its media type; all other event attributes MUST be mapped to the Kafka message's header section.

* https://quarkus.io/blog/kafka-cloud-events/[Quarkus: Sending and Receiving Cloud Events with Kafka], also very good article, selected quotes below
** `id` is a string identifies the event. This is able to be UUID or counter and must have a unique value.
** The `source` attribute is a URI identifying the event source, i.e., the context in which an event happened or the application that emitted that specific event. Combination of source and id must be unique all across the system.
** `type` is event data type used by the source
** The other attributes are optional. datacontenttype defines the content-type of the data attribute. subject allows passing extra details about the event, such as an additional hint about the context or the type of event. time is a timestamp, generally indicating the creation time. There is another optional attribute not used in my example. The dataschema attribute lets you pass the schema of the event data.
* https://cloudevents.github.io/sdk-java/json-jackson.html[CloudEvents Json Jackson] for JSON encoded event data
* Code Examples
** https://github.com/cloudevents/sdk-java/blob/main/examples/kafka/src/main/java/io/cloudevents/examples/kafka/SampleProducer.java[SampleProducer source including encoding config]
** https://github.com/cloudevents/sdk-java/blob/main/examples/kafka/src/main/java/io/cloudevents/examples/kafka/SampleConsumer.java[SampleConsumer source including decoding config]
